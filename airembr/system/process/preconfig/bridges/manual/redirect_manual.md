# Link Redirect Guide

**Web Traffic Redirects** let you redirect users from a custom link to a specific URL, while also sending an observation 
to the System for tracking. This is useful for tracking link clicks in emails, websites, or campaigns.

When someone clicks your redirect link:

* They’re sent to your chosen URL.
* An event is triggered and sent to the System, including optional data (like customer ID or source info).

---

## How to Set It Up

Go to **Source → Event Redirects** and click **Redirect URL Bridge** to create a new redirect.

### 1. Fill Out Basic Information

* **Name**: A name to identify this redirect (e.g., **Promo Link Click**)
* **Channel** *(optional)*: Where the traffic is from (e.g., **email**, **web**)
* **Description** *(optional)*: A short note (e.g., **Tracks clicks on promo email links**)
* **Enabled**: Check this to activate the redirect

### 2. Configure the Redirect

In the **Redirect Bridge Configuration** section:

* **Event Type**: Name of the event to trigger (e.g., **promo-click**)

* **Customer ID URL Param**: URL parameter used to extract customer ID (e.g., **cid**)

* **Event Properties (JSON)**: Add any extra event data. Use **payload@paramName** to refer to URL parameters.

  ******json
  {
    "source": "email-campaign",
    "campaign_id": "spring2025",
    "user_id": "payload@cid"
  }
  ******

* **Redirect URL**: The final destination users should be taken to (e.g., **https://yourdomain.com/offer**)

---

## Redirect Link Format

After saving, your redirect link will look like this:

******
http://<api-url>/redirect/<redirect-id>
******

* Replace **<api-url>** with your System server URL
* **<redirect-id>** is auto-generated by System and identifies your redirect

You can find and manage your redirect links under **Source → Event Redirects**.
